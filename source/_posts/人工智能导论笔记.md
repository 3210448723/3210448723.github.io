<li><p>符号主义(Symbolicism)&#x2F;逻辑主义(Logicism)&#x2F;心理学派(Psychlogism)&#x2F;计算机学派(Computerism)</p>
<blockquote>
<p>其原理主要为物理符号系统(即符号操作系统)假设和有限合理性原理。</p>
</blockquote>
<ul>
<li>认为人的认知基元是符号，而且认知过程即符号操作过程。</li>
<li>认为人是一个物理符号系统，计算机也是一个物理符号系统，因此，我们就能够用计算机来模拟人的智能行为。</li>
<li>认为知识是信息的一种形式，是构成智能的基础。</li>
<li>认为人工智能的核心问题是知识表示、知识推理和知识运用。</li>
</ul>
</li>

<h2 id="问题归约表示"><a href="#问题归约表示" class="headerlink" title="问题归约表示"></a>问题归约表示</h2><p>基于状态空间</p>
<h3 id="与状态空间表示的关系"><a href="#与状态空间表示的关系" class="headerlink" title="与状态空间表示的关系"></a>与状态空间表示的关系</h3><h4 id="PPT"><a href="#PPT" class="headerlink" title="PPT"></a>PPT</h4><ol>
<li>状态空间法是一种基于解答空间的问题表示和求解方法，它是以状态和操作符为基础的。在利用状态空间图表示时,从某个初始状态开始,每次加一个操作符,递增地建立起操作符的试验序列,直到达到目标状态为止.由于<strong>状态空间法</strong>需要扩展过多的节点,容易出现“<strong>组合爆炸</strong>“,因而只适用于表示比较简单的问题。</li>
<li>问题归约法从目标（要解决的问题）出发,逆向推理,通过一系列变换把初始问题变换为子问题集合和子子问题集合,直至最后归约为一个平凡的本原问题集合.这些本原问题的解可以直接得到,从而解决了初始问题,用与或图来有效地说明问题归约法的求解途径。</li>
<li>状态空间法是问题归约法的一种<strong>特例</strong>。在问题归约法的与或图中,包含有<strong>与节点和或节点</strong>，而在状态空间法中只含有<strong>或节点</strong>。</li>
</ol>
<h4 id="chatGPT-4"><a href="#chatGPT-4" class="headerlink" title="chatGPT-4"></a>chatGPT-4</h4><ol>

<li><p>问题归约表示关注于将一个复杂问题分解为若干个简单的子问题，然后通过解决这些子问题来解决原问题。</p>
</li>
<li><p>状态空间表示则关注于描述问题的所有可能状态以及从一个状态转移到另一个状态的规则，从而找到从初始状态到目标状态的路径。</p>

<li><p>表示方法</p>
<ol>
<li><p>问题归约表示通常使用树形结构来表示问题的分解过程，每个节点表示一个子问题，边表示子问题之间的关系。</p>
</li>
<li><p>状态空间表示使用图形结构来表示问题的所有可能状态，节点表示状态，边表示状态之间的转换。</p>
</li>
</ol>
</li>
<li><p>求解策略</p>
<ol>
<li><p>问题归约表示通常采用自顶向下的求解策略，从原问题开始，逐步分解为子问题，直到子问题可以直接解决。</p>
</li>
<li><p>状态空间表示通常采用搜索算法（如深度优先搜索、广度优先搜索、A*搜索等）来在状态空间中寻找从初始状态到目标状态的路径。</p>
</li>
</ol>
</li>
<li><p>适用场景</p>
<ol>
<li>问题归约表示适用于可以分解为相互独立或相对简单的子问题的问题，如排序、最短路径等。</li>
<li>状态空间表示适用于需要在大量可能状态中寻找解决方案的问题，如八皇后问题、旅行商问题等。</li>
</ol>
</li>
</ol>
<h3 id="实质"><a href="#实质" class="headerlink" title="实质"></a>实质</h3><p>从目标(要解决的问题)出发逆向推理，建立子问题以及子问题的子问题，直至最后把初始问题归约为一个平凡的本原问题集合。</p>

<li><p>与或图中的每个节点代表一个要解决的单一问题或问题集合。</p>
<p>起始节点对应于原始问题。终叶节点对应于本原问题的节点。</p>

<h4 id="与节点"><a href="#与节点" class="headerlink" title="与节点"></a>与节点</h4><p>只有解决所有子问题，才能解决其父辈问题的节点集合，如（B，C）和（D，E，F）。各个节点之间用一段小圆弧连接标记。</p>
<h4 id="可解节点"><a href="#可解节点" class="headerlink" title="可解节点"></a>可解节点</h4><ul>
<li>终叶节点是可解节点(因为它们与本原问题相关联)。</li>
<li>如果某个非终叶节点含有<strong>或</strong>后继节点，那么<strong>只要有一个</strong>后继节点是<strong>可解</strong>的时，此非终叶节点就是可解的。</li>
<li>如果某个非终叶节点含有<strong>与</strong>后继节点，那么只有其<strong>全部</strong>后继节点为<strong>可解</strong>时，此非终叶节点才是可解的。</li>
</ul>

<p>实心节点有解，空心节点无解；标了t的节点是终叶节点，没有标的是非终叶节点。</p>
</blockquote>
<h4 id="不可解节点"><a href="#不可解节点" class="headerlink" title="不可解节点"></a>不可解节点</h4><ul>
<li><strong>没有后裔的非终叶节点</strong>为不可解节点。</li>
<li>如果某个非终叶节点含有<strong>或</strong>后继节点，那么只有当其<strong>全部</strong>后裔为<strong>不可解</strong>时，此非终叶节点才是不可解的。</li>
<li>如果某个非终叶节点含有<strong>与</strong>后继节点，那么只要当其后裔有<strong>一个</strong>为<strong>不可解</strong>时，此非终叶节点就是不可解的。</li>
</ul>

<h5 id="合式公式"><a href="#合式公式" class="headerlink" title="合式公式"></a>合式公式</h5><h6 id="递归定义"><a href="#递归定义" class="headerlink" title="递归定义"></a>递归定义</h6><ul>
<li>原子谓词公式是合式公式。</li>
<li>若A为合式公式，则～A也是一个合式公式。</li>

<li>若A是合式公式，x为A中的自由变元，则(∀ x)A和(ョ x)A都是合式公式。</li>
<li>只有按上述规则求得的那些公式，才是合式公式。</li>
</ul>

<ol>

<li>设置一个通向n的指针</li>
<li>将它们加进OPEN表</li>
</ol>
</li>
<li>对已经在OPEN或CLOSED表上的每个M成员，确定是否需要将其原来的父节点改为 n</li>
<li>对已在CLOSED表上的每个M成员，若修改了其父节点，则将该节点从 CLOSED 表中移出，重新加入 OPEN 表中。</li>
</ol>
</li>
<li><p>按某一任意方式或按某个试探值，重排OPEN表。</p>
</li>
<li><p>GO <strong>LOOP</strong>。</p>
</li>
</ol>
</li>
<li><p>重排OPEN表意味着，在第(6)步中，将优先扩展哪个节点，<strong>不同的排序标准对应着不同的搜索策略</strong>。</p>
</li>
<li><p>重排的原则当视具体需求而定，不同的原则对应着不同的搜索策略</p>
<ol>
<li>如果想尽快地找到一个解，则应当将<strong>最有可能</strong>达到目标节点的那些节点排在OPEN表的前面部分</li>
<li>如果想找到代价最小的解，则应当按<strong>代价从小到大</strong>的顺序重排OPEN表。</li>
</ol>
</li>
</ol>

<h3 id="基本概念"><a href="#基本概念" class="headerlink" title="基本概念"></a>基本概念</h3><h4 id="个体"><a href="#个体" class="headerlink" title="个体"></a>个体</h4><ul>
<li>模拟生物个体而对问题中的对象（一般就是问题的解）的一种称呼</li>
<li>一个个体也就是搜索空间中的一个点</li>

<h4 id="种群-population"><a href="#种群-population" class="headerlink" title="种群(population)"></a>种群(population)</h4><ul>
<li>模拟生物种群而由若干个体组成的群体</li>
<li>一般是整个搜索空间的一个很小的子集</li>
<li>通过对种群实施遗传操作，使其不断更新换代而实现对整个论域空间的搜索</li>
</ul>
<h4 id="适应度-fitness"><a href="#适应度-fitness" class="headerlink" title="适应度(fitness)"></a>适应度(fitness)</h4><p>借鉴生物个体对环境的适应程度，而对问题中的个体对象所设计的表征其优劣的一种测度</p>

<li>问题中的全体个体与其适应度之间的一个对应关系</li>
<li>一般是一个实值函数，该函数就是遗传算法中指导搜索的评价函数</li>
</ul>

<li>生物学：染色体是由若干基因组成的位串</li>
<li>遗传算法<ul>
<li>染色体是问题中个体的某种字符串形式的编码表示</li>
<li>染色体以字符串来表示</li>
<li>基因是字符串中的一个个字符</li>
</ul>
</li>

</ul>

<li><p>原理</p>
<p><img src="https://img-blog.csdnimg.cn/0974213befff42ce9b8538990450cc73.jpeg" alt="image-20230506164241989"></p>
<ol>
<li>做一个单位圆，然后按各个染色体的选择概率将圆面划分为相应的扇形区域</li>
<li>转动轮盘，轮盘静止时指针指向某一扇区，即为选中扇区，相应的个体&#x2F;染色体即被选中</li>

<li><p>若r≤q<del>1</del>，则染色体1被选中</p>
</li>
<li><p>若q<del>k-1</del>&lt; r ≤q<del>k</del>(2≤k≤N)，则染色体k被选中</p>
<p>q<del>i</del>为染色体x<del>i</del>(i&#x3D;1, 2, …, n)的累积概率</p>
</li>
</ol>

<h3 id="遗传算法"><a href="#遗传算法" class="headerlink" title="遗传算法"></a>遗传算法</h3><h4 id="定义-9"><a href="#定义-9" class="headerlink" title="定义"></a>定义</h4><ul>
<li>遗传算法是模仿生物遗传学和自然选择机理，通过人工方式所构造的一类优化搜索算法，是对生物进化过程进行的一种数学仿真，是进化计算的最重要的形式。</li>
<li>遗传算法为那些难以找到传统数学模型的难题指出了一个解决方法。</li>
</ul>
<h4 id="参数"><a href="#参数" class="headerlink" title="参数"></a>参数</h4><ol>
<li>种群规模：种群的大小，用染色体个数表示</li>
<li>最大换代数：种群更新换代的上限，也是算法终止一个条件</li>
<li>交叉率Pc：参加交叉运算的染色体个数占全体染色体总数的比例取值范围：0.4-0.99</li>
<li>变异率Pm<ol>
<li>发生变异的基因位数占全体染色体的基因总位数的比例</li>
<li>取值范围：0.0001-0.1</li>
</ol>
</li>
<li>染色体编码：长度L</li>
</ol>
<h4 id="基本原理✦"><a href="#基本原理✦" class="headerlink" title="基本原理✦"></a>基本原理✦</h4><ul>
<li>通过<strong>随机</strong>方式产生若干个所求解问题的数字编码（染色体），形成<strong>初始群体</strong>；</li>
<li>通过<strong>适应度</strong>函数给每个个体一个<strong>数值评价</strong><ul>
<li><strong>淘汰低</strong>适应度的个体</li>
<li><strong>选择高</strong>适应度的个体参加遗传操作（交叉、变异）</li>
</ul>
</li>
<li>经过遗传操作后的个体集合形成<strong>下一代</strong>新的种群。对这个新种群进行下一轮进化。</li>
</ul>
<h4 id="遗传算法步骤✦"><a href="#遗传算法步骤✦" class="headerlink" title="遗传算法步骤✦"></a>遗传算法步骤✦</h4><ol>
<li><strong>初始化</strong>群体;</li>
<li>计算群体上每个个体的<strong>适应度</strong>值;</li>
<li>按由个体适应度值所决定的某个规则<strong>选择</strong>将进入下一代的个体;</li>
<li>按概率Pc进行<strong>交叉</strong>操作;</li>
<li>按概率Pm进行<strong>突变</strong>操作;</li>
<li>若没有满足某种<strong>停止条件</strong>，则转第(2)步，否则进入下一步。</li>
<li><strong>输出</strong>群体中<strong>适应度值最优</strong>的染色体作为问题的满意解或最优解。</li>
</ol>

<li>每只鸟抽象为一个无质量，无体积的“粒子”</li>
<li>每个粒子有一个速度决定它们的飞行方向和距离，初始值可随机确定</li>
<li>每一次单位时间的飞行后，所有粒子分享信息，下一步将飞向自身最佳位置和<strong>全局或邻域</strong>最优位置的加权中心</li>
<li>每次迭代中，粒子通过跟踪“个体极值”和“全局极值”来更<br>新自己的位置</li>
</ol>
<h4 id="算法流程"><a href="#算法流程" class="headerlink" title="算法流程"></a>算法流程</h4><ol>
<li>初始化一群粒子（群体规模为m），包括随机的位置和速度；</li>
<li>评价每个粒子的适应度；</li>
<li>对每个粒子，更新个体最优位置；</li>
<li>更新全局最优位置；</li>
<li>根据速度和位置更新方程更新粒子速度和位置；</li>
<li>如未达到结束条件（通常为足够好的适应值或达到一个预设最大迭代次数） ，回到2。</li>
</ol>

<li>多模态学习：让AI系统能够处理和理解来自多种感官（如视觉、听觉、触觉等）的信息。这有助于开发更加智能和<br>自适应的AI系统。</li>
<li>元学习（Meta-learning）：通过让AI系统自我学习如何学习，以提高模型在新任务和领域的泛化能力。</li>
<li>小样本学习：研究如何在少量标注数据的情况下，让机器学习模型实现高效的学习。这有助于降低AI系统对大量标<br>注数据的依赖。</li>
<li>安全AI：研究如何确保AI系统在各种条件下的安全性、稳定性和可控性，包括抵御对抗性攻击、防止模型崩溃等。</li>
<li>人工智能与心理学、神经科学的交叉研究：通过模仿人类大脑和认知过程来发展AI系统，以提高其智能和适应能力。</li>
</ol>
